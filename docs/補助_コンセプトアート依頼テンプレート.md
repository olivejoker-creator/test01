# 補助_コンセプトアート依頼テンプレート（汎用）

> **役割**：ChatGPTにコンセプトアートを依頼するための汎用テンプレート（STEP3）。
> どのMODでも使えるよう、対象（アイテム/ブロック/Mob/装備等）やスタイル（リアル寄り/デフォルメ寄り）を選べる形にしている。
> 【】の箇所を埋めて使う。1対象につき1回依頼する（複数を一度に頼まない）。

---

## 0. 最初に決める：絵柄スタイル（このMOD共通の方針）

そのMODで作る絵の方向性を、最初に1つ決めて固定する（01企画書にも記載）。

| スタイル | 説明 | 向くMOD |
|---|---|---|
| A. バニラ調（デフォルメ） | Minecraftの標準的なボクセル/キューブ調。世界に溶け込む | バニラ寄りの追加要素 |
| B. やや写実（セミリアル） | デフォルメは保ちつつ、質感・陰影を少し豊かに | 装備・宝石・機械など |
| C. リアル寄り | 質感・形状を写実的に。資料性重視 | 独自世界観・大型Mob |

> 迷ったら **A（バニラ調）** が無難。Minecraftに馴染みやすい。
> 決めたスタイルは、下のテンプレの【スタイル】に毎回書く。

---

## 1. 基本テンプレート（1対象ぶん・コピーして使う）

```
Minecraft 【ローダー】 【MCバージョン】 向けMOD「【MOD名】」のコンセプトアートを制作してください。

【対象の種類】アイテム / ブロック / Mob / 装備・道具 / GUI / その他（いずれか）
【今回の対象】【ID または 名前（English Name）】

【コンセプト】（1〜2行。それが何で、どんな役割・性格・雰囲気か）
【能力・用途】（効果・使い方・特殊な挙動など。Mobなら行動や生息地）
【世界観】（登場する場面・バイオーム・背景設定など）

【スタイル】（A:バニラ調 / B:セミリアル / C:リアル寄り のどれか。上で決めた方針）
【見た目の方向性】
-（モチーフの説明。実在のもの・テーマなど）
-（配色の方向性。基調色・アクセント）
-（形・シルエットの特徴。個性が出る部位）
-（効果・用途が視覚的に伝わる要素があれば）

【画像の要件】
- この1対象のみを描く（他のものを同じ画像に入れない）
- 背景は白または単色
- 全体が見える
- 正面・背面・側面・斜めなど複数方向が確認できること（立体物の場合）
- 選んだ【スタイル】に沿った表現にすること
- テクスチャ制作・色彩判断・3Dモデル制作の参考資料として使えること
```

---

## 2. 状況別の追加指定（必要なものだけ足す）

### アイテム／装備（小さなドット絵になる想定）
```
- 最終的に【解像度（例:16×16）】のテクスチャに落とし込める、シンプルで読みやすい形にする
- インベントリアイコンとして小さくても判別できるシルエットにする
```

### ブロック
```
- 6面（上面・側面・底面）が分かるように描く
- 既存のバニラブロックと並べても違和感のない質感・解像度にする
```

### 発光する対象（目・体の一部が光る）
```
- 重要:（部位）が（色）に光る（発光する）。暗い背景でぼうっと光る印象を強調
- 暗い背景で発光の様子が分かる図を1つ添えてもよい（発光表現の参考用）
```

### 2状態を持つ対象（通常/特殊状態の切り替え）
```
-（対象）には「通常状態」と「（特殊）状態」の2パターンがある
- 両方を描き、違いが明確に分かること
```

### 小型/大型など体格・サイズが特徴
```
- 標準（バニラの基準物）より（小さい/大きい）ことが分かる外観
- 大きさ比較を添える
```

### 専用モデル（複数形態・段階成長・特殊シルエット）
```
- この1対象のみを扱う
-（形態/段階）それぞれを描き、名称を明記する
- 各（形態/段階）が全体で確認できる
- シルエットの違いが明確であること（これが最重要）
```

### 騎乗できるMob
```
- 鞍（サドル）を乗せて騎乗するイメージ図を添える
-（滑空など特殊移動があれば）そのイメージ図も添える
```

### 白基調など背景に埋もれやすい対象
```
- 背景は白または単色（白い対象の場合、輪郭や影で見やすくすること）
```

---

## 3. 依頼の進め方（運用ルール）

1. **1対象ずつ依頼する**（複数を一度に頼まない）
2. 出てきたコンセプトを確認・レビュー（コンセプト整合・能力との矛盾・要件・実装可能性）
3. 承認 → 次の対象へ。修正が必要なら具体的に指摘して再依頼
4. 採用した画像は、必要なら `concept/`（任意）に保存して後の工程の参考にする

## 4. 画像生成の共通ルール（厳守）

- 1画像につき1対象のみ
- 背景は白または単色
- 全体が見える
- 立体物は複数方向（正面・背面・側面・斜め等）
- 選んだスタイルに沿った表現
- テクスチャ作成・色彩判断・モデル制作の参考資料として使用可能
- 複数の対象を1枚にまとめてはならない

## 5. 記入例（バニラ調・アイテムの例）

```
Minecraft Forge 1.20.1 向けMOD「Ruby Tools」のコンセプトアートを制作してください。

【対象の種類】アイテム
【今回の対象】ruby（Ruby）

【コンセプト】序盤を支える新鉱石「ルビー」。鉄とダイヤの間の素材。
【能力・用途】道具や装備の素材。村人との取引にも使える。
【世界観】地中（y0〜16）でまれに採れる。

【スタイル】A:バニラ調（デフォルメ）
【見た目の方向性】
- 宝石モチーフ。深い赤と、宝石らしい面のきらめき
- バニラのエメラルド/ダイヤと並べても馴染む質感
- 16×16でも「赤い宝石」と一目で分かるシルエット

【画像の要件】
- この1対象のみを描く
- 背景は白または単色
- 全体が見える
- 最終的に16×16テクスチャに落とし込める形
- テクスチャ制作・色彩判断の参考資料として使えること
```

## 6. 記入例（リアル寄り・Mobの例）

```
Minecraft Forge 1.20.1 向けMOD「Unique Chickens」のコンセプトアートを制作してください。
バニラのニワトリをベースにしたオリジナルのMobです。

【対象の種類】Mob
【今回の対象】UC-01 烏骨鶏（Silkie）

【コンセプト】癒やしの鶏。穏やかで温和。
【能力・用途】産んだ卵が周囲を少し回復。産卵間隔は遅い。手懐け不可。
【世界観】森に生息。

【スタイル】C:リアル寄り（資料性重視）
【見た目の方向性】
- バニラのニワトリの体型がベース
- とさか・尾羽・冠羽などの部位で個性を出す
- 烏骨鶏（もこもこした品種）と「癒やし」が伝わる外観

【画像の要件】
- この1種のみを描く
- 背景は白または単色
- 全身が見える
- 正面・背面・側面・斜めの4方向
- テクスチャ・3Dモデル制作の参考資料として使えること
```
