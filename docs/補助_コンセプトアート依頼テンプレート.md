# 補助_コンセプトアート依頼テンプレート（汎用）

> **役割**：ChatGPTにコンセプトアートを依頼するための汎用テンプレート（STEP3）。
> どのMODでも使えるよう、対象（アイテム/ブロック/Mob/装備等）やスタイル（リアル寄り/デフォルメ寄り）を選べる形にしている。
> 【】の箇所を埋めて使う。1対象につき1回依頼する（複数を一度に頼まない）。

---

## 0. 最初に決める：絵柄スタイル（このMOD共通の方針）

そのMODで作る絵の方向性を、最初に1つ決めて固定する（01企画書にも記載）。
**基本は「既存のMinecraft Mob・アイテムと並べて違和感のないデザイン」を目指す**（リアルに寄せすぎない）。

| スタイル | 説明 | 向くMOD |
|---|---|---|
| **A. バニラ準拠（推奨）** | 既存のMinecraft Mob／アイテムと同じデザインルール。ボクセル/キューブ調、低彩度、太めの輪郭、簡素なディテール。バニラに混ぜても気づかないレベル | ほぼすべてのMOD。まず迷ったらこれ |
| B. バニラ＋ディテール | バニラの形・色調は守りつつ、模様や陰影を少しだけ足す（やり過ぎない） | 特徴を少し強調したいMob・装備 |
| C. リアル寄り（非推奨・限定） | 質感・形状を写実的に。**バニラから浮くので原則使わない**。資料用に細部を見たいときだけ | 独自世界観で意図的に浮かせる場合のみ |

> **原則 A（バニラ準拠）**。以前の「セミリアル」はリアルに寄りすぎてバニラから浮いたため、
> 既存Mobと同じデザインルールに合わせる方針へ変更。
> Cはバニラから浮くので、明確な理由がない限り選ばない。
> 決めたスタイルは、下のテンプレの【スタイル】に毎回書く。

**バニラ準拠のデザインルール（A・Bで守る）**
- 体型・パーツは直方体（ボックス）ベース。曲面や細かい突起を作り込みすぎない
- 配色は低〜中彩度（ビビッドにしすぎない）。バニラの色味に寄せる
- テクスチャは16×16基準のドット感を残す（なめらかなグラデを多用しない）
- 輪郭・面の区切りははっきりと。小さくても形が読める
- 既存の近いバニラMob/アイテム（例：ニワトリ、ウシ、鉄インゴット等）を「並べても馴染むか」で判断

## 1. 基本テンプレート（1対象ぶん・コピーして使う）

```
Minecraft 【ローダー】 【MCバージョン】 向けMOD「【MOD名】」のコンセプトアートを制作してください。

【対象の種類】アイテム / ブロック / Mob / 装備・道具 / GUI / その他（いずれか）
【今回の対象】【ID または 名前（English Name）】
【アセット名（ファイル名）】【種別_番号2桁_英名（例：mob_01_tyrannosaurus）。画像は concept/<アセット名>.png で保存】

【コンセプト】（1〜2行。それが何で、どんな役割・性格・雰囲気か）
【能力・用途】（効果・使い方・特殊な挙動など。Mobなら行動や生息地）
【世界観】（登場する場面・バイオーム・背景設定など）

【スタイル】（A:バニラ準拠 / B:バニラ＋ディテール / C:リアル寄り のどれか。原則A。上で決めた方針）
【見た目の方向性】
-（モチーフの説明。実在のもの・テーマなど）
-（形・シルエットの特徴。個性が出る部位）
-（効果・用途が視覚的に伝わる要素があれば）
-（既存のどのバニラMob/アイテムに近いか。並べて馴染む基準）

【カラーパレット】（バニラの色味に寄せた低〜中彩度で。各色おおよそのHEXも添える）
- 基調色：（例：くすんだ赤 #A03A2C）
- 副色：（例：濃い茶 #5A2E1B）
- アクセント：（例：淡い黄 #E8C95A／使うなら1〜2色まで）
- 影/暗部：（例：暗赤 #6E2218）
※色数は絞る（3〜5色目安）。ビビッドにしすぎない／バニラのパレットと並べて浮かないこと

【画像の要件】
- この1対象のみを描く（他のものを同じ画像に入れない）
- 背景は白または単色
- 全体が見える
- 正面・背面・側面・斜めなど複数方向が確認できること（立体物の場合）
- 選んだ【スタイル】に沿った表現にすること（原則バニラ準拠＝ボックス基調・16×16のドット感・低彩度）
- 上記カラーパレットの色だけで塗り、配色表（色見本＋HEX）も画像内か別途で示すこと
- テクスチャ制作・色彩判断・3Dモデル制作の参考資料として使えること
```

## 2. 状況別の追加指定（必要なものだけ足す）

### アイテム／装備（小さなドット絵になる想定）
```
- 最終的に【解像度（例:16×16）】のテクスチャに落とし込める、シンプルで読みやすい形にする
- インベントリアイコンとして小さくても判別できるシルエットにする
```

### ブロック
```
- 6面（上面・側面・底面）が分かるように描く
- 既存のバニラブロックと並べても違和感のない質感・解像度にする
```

### 発光する対象（目・体の一部が光る）
```
- 重要:（部位）が（色）に光る（発光する）。暗い背景でぼうっと光る印象を強調
- 暗い背景で発光の様子が分かる図を1つ添えてもよい（発光表現の参考用）
```

### 2状態を持つ対象（通常/特殊状態の切り替え）
```
-（対象）には「通常状態」と「（特殊）状態」の2パターンがある
- 両方を描き、違いが明確に分かること
```

### 小型/大型など体格・サイズが特徴
```
- 標準（バニラの基準物）より（小さい/大きい）ことが分かる外観
- 大きさ比較を添える
```

### 専用モデル（複数形態・段階成長・特殊シルエット）
```
- この1対象のみを扱う
-（形態/段階）それぞれを描き、名称を明記する
- 各（形態/段階）が全体で確認できる
- シルエットの違いが明確であること（これが最重要）
```

### 騎乗できるMob
```
- 鞍（サドル）を乗せて騎乗するイメージ図を添える
-（滑空など特殊移動があれば）そのイメージ図も添える
```

### 白基調など背景に埋もれやすい対象
```
- 背景は白または単色（白い対象の場合、輪郭や影で見やすくすること）
```

## 3. 依頼の進め方（運用ルール）

1. **1対象ずつ依頼する**（複数を一度に頼まない）
2. 出てきたコンセプトを確認・レビュー（コンセプト整合・能力との矛盾・要件・実装可能性）
3. 承認 → 次の対象へ。修正が必要なら具体的に指摘して再依頼
4. 採用した画像は、必要なら `concept/`（任意）に保存して後の工程の参考にする

## 4. 画像生成の共通ルール（厳守）

- 1画像につき1対象のみ
- 背景は白または単色
- 全体が見える
- 立体物は複数方向（正面・背面・側面・斜め等）
- 選んだスタイルに沿った表現（原則バニラ準拠：ボックス基調・16×16のドット感・低彩度）
- 既存のバニラMob/アイテムと並べて浮かないこと（リアルに寄せすぎない）
- カラーパレットは色数を絞り（3〜5色目安）、バニラの色味に寄せる。配色表（色見本＋HEX）も示す
- テクスチャ作成・色彩判断・モデル制作の参考資料として使用可能
- 複数の対象を1枚にまとめてはならない
