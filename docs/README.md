# Minecraft MOD制作テンプレート一式

Minecraft MOD制作を **毎回同じ手順・同じ成果物・同じ管理方法** で進めるための標準セット。
担当するAI（ChatGPT / Claude / Claude Code / Codex）や人が変わっても同じ流れで開発できることを目的とする。
Forge / NeoForge / Fabric いずれにも使える汎用フロー。

---

## ファイル構成と番号体系

**`docs/` に入れる成果物（00〜08）**：制作の本体。番号がそのまま作成順。

| 番号 | ファイル | 主担当 | 内容 |
|---|---|---|---|
| 00 | 00_プロジェクト基盤定義.md | あなた | modID・対応バージョン・ローダー等の土台 |
| 01 | 01_企画書.md | あなた / ChatGPT | 何を作るか |
| 02 | 02_開発ルール.md | （固定） | 開発ルール・リポジトリ構成・2台運用 |
| 03 | 03_仕様書.md | Claude | 実装可能な仕様 |
| 04 | 04_実装計画.md | Claude | タスク分解 |
| 05 | 05_アセット確認書.md | Claude | アセット確認（1回目の報告。実装前の素材レビュー） |
| 06 | 06_実装報告.md | Claude Code | 実装＋統合の結果（2回目の報告） |
| 07 | 07_デバッグ報告.md | Codex | 不具合と修正の記録（実装レビュー） |
| 08 | 08_完成レビュー.md | あなた / ChatGPT | リリース可否の最終確認 |

**補助テンプレート（`補助_` 接頭辞）**：作業を助ける道具。番号体系には含めない。

- 補助_コンセプトアート依頼テンプレート.md（見た目を決める／ChatGPTへの依頼文。スタイルA/B/Cを選ぶ）
- 補助_コンセプトアートチェックリスト.md（対象一覧＋進捗管理。工程2でClaudeが依頼文と一緒に作成）
- 補助_アセット制作指示書.md（素材を作る）
- 補助_タスク.md（個別タスク記入用）
- 補助_テスト計画.md
- 補助_最終確認チェックリスト.md
- 補助_GitHub運用ルール.md
- 補助_ClaudeCode標準指示書.md
- .github/（Issue・PRテンプレ）

## 開発フロー全体図（工程0〜8）

> 重要：**先に見た目（コンセプト・アセット）を固めてから実装する**流れ。
> コンセプトは2段階：Claudeが依頼文を1種1ファイルで作成 → ChatGPTが1種ずつ描画。
> 素材をClaudeが確認（1回目の報告）→ その素材で実装＋統合 → Codexが確認（2回目の報告）。

| 工程 | 内容 | 主担当 | 成果物 |
|---|---|---|---|
| 0 | 企画（資料準備） | あなた / ChatGPT | 00・01（・02） |
| 1 | 仕様書・実装計画 | Claude | 03・04 |
| 2 | コンセプト依頼文づくり | Claude | 依頼文(1種1ファイル)＋チェックリスト |
| 3 | コンセプトアート作成 | ChatGPT | 画像（1種ずつ） |
| 4 | アセット制作 | あなた / Blockbench | 素材 |
| 5 | アセット確認（1回目の報告） | Claude | 05 |
| 6 | 実装＋統合（2回目の報告） | Claude Code | コード・06 |
| 7 | 実装レビュー／デバッグ | Codex | 07 |
| 8 | 完成レビュー | ChatGPT | 08 |

> AIの役割：ChatGPT＝企画/絵/最終レビュー、Claude＝仕様づくり・コンセプト依頼文・アセット確認、Claude Code＝実装とpush、Codex＝コードのレビュー・デバッグ。
> コンセプトは「Claudeが依頼文を作る（テキストなので混ざらない）→ ChatGPTが1種ずつ描く（画像は混ざるので1種厳守）」の2段構え。
> 報告は2回：**05アセット確認（Claude）**と、実装後の **07デバッグ報告（Codex）**。

## 使い方（新しいmodを始めるとき）

1. GitHubで空リポジトリを作成（public推奨）→ クローン
2. リポジトリ直下に「入れ物」を作る（`02_開発ルール` のリポジトリ構成どおり：docs/ blockbench/ textures/ animations/ .gitignore README）
3. このテンプレの **00・01・02** を `docs/` に置く（ファイル名から「テンプレート」等は付けず、`00_プロジェクト基盤定義.md` の形にする）
4. **00（基盤定義）と 01（企画）の中身を埋める** ← まずここ
5. **仕様書・実装計画（03・04）は Claude に依頼**（「📋 00+01+02をコピー（Claude仕様書用）」を使い、相談して固める）
6. 以降この順で進める：**コンセプト依頼文づくり（Claudeが依頼文＋チェックリストを作成）→ コンセプトアート作成（ChatGPTが1種ずつ描画）→ アセット制作 → アセット確認（Claudeが05）→ 実装＋統合（Claude Codeが06）→ デバッグ（Codexが07）→ 完成レビュー（08）**

> modごとに変えるのは中身（modID・企画・仕様）だけ。フォルダ構成とルールは毎回同じ。
> 報告は2回：**05 アセット確認（Claude）**と、実装後の **07 デバッグ報告（Codex）**。

## 最小構成クイックパス（初心者向け）

1. 01企画書を1行でも埋める（例：「ルビーを追加」）
2. 00基盤定義の modID・MCバージョン・ローダーだけ決める
3. Claude Code に 03仕様書・04実装計画 → 実装まで依頼
4. 開発環境で起動して動作確認
5. 補助_最終確認チェックリストで確認 → 完成

慣れたら、コンセプトアート・アセット・テスト・デバッグを足してフル運用へ。
