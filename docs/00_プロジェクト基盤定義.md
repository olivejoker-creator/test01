# 00_プロジェクト基盤定義

> **このファイルの役割**：MOD全体の「変えてはいけない土台」を最初に固定する。
> ここで決めた値（特に modID・パッケージ名・対応バージョン）は後から変えると全コードに影響するため、**最初に確定**する。
> 記入後は Claude / Claude Code に渡し、仕様書・実装の前提資料として使う。

---

## 1. 基本情報

| 項目 | 内容 |
|---|---|
| MOD名（表示名） | （例：Ruby Tools） |
| modID（半角英小文字・数字・`_`のみ） | （例：`rubytools`） |
| 作者名 | （ここに記入） |
| ライセンス | （例：MIT / All Rights Reserved） |
| 一言説明 | （例：ルビー鉱石と道具を追加する小型MOD） |

> modIDは一度決めたら変えない。すべてのアセット・データのフォルダ名（`assets/<modid>/`）に使われる。

---

## 2. 動作環境（ここが決まらないと src/ を作れない）

| 項目 | 内容 |
|---|---|
| 対応Minecraftバージョン | （例：1.20.1） |
| ローダー | Forge / NeoForge / Fabric（いずれか1つ） |
| ローダーバージョン | （例：Forge 47.2.0） |
| Javaバージョン | （例：Java 17） |
| Mappings（必要なら） | （例：official / parchment） |

> **重要**：MCバージョン・ローダー・Javaは相互に依存する。分からない場合は「対応MCバージョン」だけ先に決め、残りはClaude Codeに「この版に合うMDKで初期化して」と任せてよい。

---

## 3. パッケージ・命名規則

| 項目 | 内容 |
|---|---|
| ルートパッケージ | （例：`com.example.rubytools`） |
| メインクラス名 | （例：`RubyTools`） |
| アセットの名前空間 | `<modid>`（上で決めたmodID） |

---

## 4. リソース配置の基準（ゲームに読み込まれる側）

ゲームが読み込む完成アセットは、必ず次の場所に置く（詳細は `02_開発ルール` のリポジトリ構成を参照）。

```text
src/main/resources/
├── META-INF/                 mods.toml 等
├── assets/<modid>/
│   ├── textures/             テクスチャ（完成品）
│   ├── models/               モデル
│   ├── lang/                 言語ファイル（en_us.json / ja_jp.json）
│   └── sounds/               音
└── data/<modid>/             レシピ・ドロップ・タグ等（DataPack）
```

> 編集中の原本（.bbmodel やテクスチャ元データ）は、リポジトリ直下の `blockbench/` `textures/` `animations/` に置く。完成品だけを上記へ配置する。

---

## 5. スコープ（やること・やらないこと）

- **このMODでやること**：（例：ルビー鉱石・ルビー・ルビー製ツール3種の追加）
- **やらないこと（今回は対象外）**：（例：Mob追加、次元追加、コンフィグGUI）

> スコープを最初に絞ると、実装・デバッグが安定する。最初は極小（アイテム1個など）から。
