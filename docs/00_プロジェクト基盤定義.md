# 00 プロジェクト基盤定義テンプレート

> **このファイルの役割**：MOD全体の「変えてはいけない土台」を最初に固定します。
> ここで決めた値（特にmodID・パッケージ名）は後から変えると全コードに影響するため、**最初に確定**してください。
> 記入後はClaudeに渡し、仕様書作成の前提資料として使います。

記入方法：`【記入】` の部分を埋めてください。例は参考です。

---

## 1. 基本情報

| 項目 | 内容 | 例 |
|---|---|---|
| MOD名（表示名） | 【記入】 | Ancient Relics |
| modID（内部ID） | 【記入】 | ancientrelics |
| パッケージ名 | 【記入】 | com.yourname.ancientrelics |
| 作者名 | 【記入】 | YourName |
| MODバージョン（初期） | 【記入】 | 0.1.0 |
| 短い説明（1行） | 【記入】 | 遺跡探索を楽しくする装備とブロックを追加 |

> **modIDのルール**：小文字・英数字とアンダースコアのみ。スペース・大文字・記号は不可。
> 一度決めたら変えないこと（リソースフォルダ名・登録名すべてに使われます）。

## 5. リソース配置の基準（このローダーでの置き場所）

実装時に迷わないよう、配置ルールを明記します（`<modid>` は上で決めた値に置換）。

```
src/main/resources/
├── META-INF/
│   └── neoforge.mods.toml          # メタデータ（Forge=mods.toml / Fabric=fabric.mod.json）
├── assets/<modid>/
│   ├── lang/                       # 言語ファイル en_us.json / ja_jp.json
│   ├── models/item/                # アイテムモデルJSON
│   ├── models/block/               # ブロックモデルJSON
│   ├── blockstates/                # ブロックステートJSON
│   ├── textures/item/              # アイテムテクスチャPNG
│   ├── textures/block/             # ブロックテクスチャPNG
│   └── sounds/                     # サウンドOGG
└── data/<modid>/
    ├── recipes/                    # クラフトレシピJSON
    ├── loot_tables/                # ドロップ定義JSON
    ├── tags/                       # タグJSON
    └── advancements/               # 進捗JSON（任意）
```
