# 04_実装計画

> **このファイルの役割**：`03_仕様書` をタスク（Task001…）に分解し、各タスクに「目的」「完了条件」を付ける。
> 実装は `02_開発ルール` の担当分担に従い、本書を受けて Claude Code が行う。
> 本書はタスクの定義であり、仕様の追加・変更は行わない（仕様変更が必要な場合は `03` を先に更新する）。

---

## 0. 前提・進め方

- 対応：Forge 1.20.1 / Java 17 / modID `dinoparadise` / パッケージ `com.ot.dinoparadise`。
- 配置は `02_開発ルール 4. リポジトリ構成` に厳守する（`src/main/resources/assets/dinoparadise/` と `data/dinoparadise/`）。
- 各タスクは原則 `dev` ブランチで作業し、開始前 `git pull` / 完了後 `git push`。コミットメッセージは規約に従う。
- 数値・挙動は `03_仕様書` の暫定値を使用し、Config 化対象は Config から読む。
- 不明点が出たら勝手に補完せず `03 9.質問事項` へ追記して確認する。

## Task001：プロジェクト基盤セットアップ

- **目的**：Forge 1.20.1 / Java17 のMOD骨格を生成し、ビルドが通る状態を作る。
- **作業**：`build.gradle` 等のビルド構成、`mods.toml`（`META-INF/`）、メインクラス `DinoParadise`、`com.ot.dinoparadise` パッケージ、`.gitignore`（build/run/.gradle 等除外）、リポジトリ標準構成のフォルダ生成。
- **完了条件**：
  - `gradlew build` が成功する。
  - 空MODとしてゲームが起動し、MOD一覧に Dino Paradise が表示される。
  - `02_開発ルール 4.` の構成と一致している。

## Task002：レジストリ登録基盤

- **目的**：Entity / Item / Block / MenuType / Feature 等の DeferredRegister 基盤を用意する。
- **作業**：各レジストリの登録クラスを作成し、メインクラスからバスに登録。
- **完了条件**：空の登録基盤でビルド・起動が通り、以降のタスクが登録物を差し込める。

## Task003：ティラノサウルス エンティティ核

- **目的**：`tyrannosaurus` エンティティを最小構成で追加する。
- **作業**：Entity クラス、属性（5-2 成体値を仮で適用）、基本AI（移動・対象探索・近接攻撃）、登録、暫定モデル/レンダラーで可視化。
- **完了条件**：
  - スポーンエッグ（または `/summon`）で出現する。
  - 移動・攻撃の基本AIが動作する。
  - クラッシュしない。

## Task004：成長システム

- **目的**：幼体→若年体→成体の段階と時間経過成長を実装する。
- **作業**：成長段階の状態保持（NBT）、段階別ステータス/サイズ/騎乗・インベントリ可否の切替（5-2）、時間経過遷移（5-3 の tick）、Config 参照。
- **完了条件**：
  - 各段階で HP・攻撃力・当たり判定が表に従って変化する。
  - 規定 tick 経過で段階が進み、成体で停止する。
  - 段階は再ログイン後も保持される。

## Task005：食性システム（犬準拠）

- **目的**：生肉での回復と成長短縮を実装する。
- **作業**：`dinoparadise:tyrannosaurus_food` タグ判定、満腹時は食べない、給餌でHP回復、幼体/若年体は成長残り時間短縮（Config ON/OFF・率）。繁殖には入らない。
- **完了条件**：
  - 生肉で回復し、満タン時は消費されない。
  - 給餌で成長が早まる（Config OFF時は早まらない）。
  - love mode に入らない。

## Task006：性別システム

- **目的**：性別決定とカーソル時表示を実装する。
- **作業**：孵化・スポーン時に♂/♀を 50/50 決定し NBT 保持、視線時HUDに名前＋性別を表示（Q-06暫定）。
- **完了条件**：
  - 個体ごとに性別が固定保持される。
  - カーソルを合わせると名前と性別が表示される。
  - 挙動に影響しない。

## Task007：所有・AI命令

- **目的**：飼い主設定と5命令（追従/待機/防衛/中立/攻撃的）を実装する。
- **作業**：飼い主UUID保持、命令状態保持、各命令AI（5-5）、「攻撃的」は敵対Mob限定（Q-07暫定）。未所有個体は敵対AIのまま。
- **完了条件**：
  - 飼い主が設定・保持される。
  - 5命令が定義通り切替・動作する。
  - 「攻撃的」で村人・動物を襲わない。

## Task008：卵ブロック・孵化

- **目的**：`tyrannosaurus_egg` ブロックと孵化を実装する。
- **作業**：設置でブロック化、設置者UUID保持、熱源タグ `incubator_heat_source` 隣接時のみ進行、規定tickで孵化（幼体出現＋ブロック消滅）、飼い主継承、卵はクラフト不可。
- **完了条件**：
  - 卵を設置でき、熱源有無で進行/停止が切替わる。
  - 孵化で幼体が出現し、設置者が飼い主になる。
  - 孵化後ブロックが消える。

## Task009：巣ブロック・自然生成

- **目的**：`tyrannosaurus_nest` コンテナと自然生成を実装する。
- **作業**：コンテナGUI（卵を取り出すと空・補充なし／Q-02暫定）、自然生成 Feature（全バイオーム低頻度・卵1〜2内包）、クラフト/設置/回収不可（Q-03暫定）。
- **完了条件**：
  - 巣が自然生成され、中に卵が入っている。
  - 卵を取り出すと空になり、補充されない。
  - クラフト・設置・回収ができない。

## Task010：ホイッスル（アイテム＋操作）

- **目的**：`dino_whistle` で個別/集団命令を出せるようにする。
- **作業**：しゃがみ＋右クリックで命令巡回（アクションバー表示）、恐竜右クリックで個別適用、空中右クリックで全所有恐竜へ適用。Task007 の命令状態と連動。
- **完了条件**：
  - 命令の巡回・表示ができる。
  - 個別/集団それぞれに命令が反映される。
  - 他人の恐竜には作用しない。

## Task011：騎乗システム

- **目的**：成体＋サドルでの騎乗・移動を実装する。
- **作業**：サドル装着時のみ騎乗可、空手右クリックで乗る、WASD＋ジャンプ移動（馬準拠）。騎乗中攻撃（左クリック噛みつき）は Config 連動で実装、難しければ本タスクは移動まで・攻撃を別途分割（Q-05）。
- **完了条件**：
  - 成体＋サドルで騎乗・移動できる。
  - サドル無し・成体未満では騎乗不可。
  - （攻撃を含む場合）左クリックで前方攻撃できる。

## Task012：恐竜インベントリGUI

- **目的**：成体の収納GUIを実装する。
- **作業**：しゃがみ＋右クリックでGUI、サドルスロット×1＋収納15（3×5）、内容のNBT保持、サドルスロット連動で騎乗可否（Task011）。
- **完了条件**：
  - 成体でGUIが開き、サドル＋15スロットが機能する。
  - サドルを入れると騎乗可能になる。
  - 収納内容が保持される。

## Task013：アイテム実装（サドル/ホイッスル/図鑑）

- **目的**：3アイテムを登録し基本挙動を付ける。
- **作業**：`tyrannosaurus_saddle`・`dino_whistle`・`dino_encyclopedia` の登録、テクスチャ/モデル雛形、用途接続（サドル→Task012、ホイッスル→Task010、図鑑→Task014）。
- **完了条件**：3アイテムが存在し、各機能の入口として動作する。

## Task014：図鑑GUI（データ駆動）

- **目的**：図鑑の閲覧GUIを実装する。
- **作業**：右クリックでGUI、1ページ1種・左右切替、表示項目（名前/食性/生息地/役割/サイズ/成長段階/解説）、恐竜データをデータ駆動で定義しティラノを登録、将来種追加可能な構造。
- **完了条件**：
  - 図鑑が開き、ティラノの全項目が表示される。
  - データ追加で種を増やせる構造になっている。
  - ロック無しで閲覧できる。

## Task015：スポーン・卵ドロップ

- **目的**：野生スポーンと討伐ドロップを実装する。
- **作業**：全バイオーム・昼夜問わず・単独・成体・敵対でスポーン（weight=2 等／Config）、討伐時5%で卵ドロップ（Config）。
- **完了条件**：
  - 野生成体が低確率でスポーンし敵対する。
  - 討伐時に設定確率で卵がドロップする。
  - 値が Config から反映される。

## Task016：レシピ・タグ

- **目的**：クラフトレシピとタグを `data/dinoparadise/` に整備する。
- **作業**：サドル/ホイッスル/図鑑のレシピ（7-1）、卵・巣はレシピ無し、`incubator_heat_source`・`tyrannosaurus_food` タグ定義（7-2）。
- **完了条件**：
  - 3アイテムがクラフトできる。
  - 卵・巣はクラフトできない。
  - 各タグが孵化/給餌判定に効いている。

## Task017：Config整備

- **目的**：`03 8.` の項目をすべて Config 化する。
- **作業**：Forge Config に各値を定義し、各タスクの参照箇所を Config 読み出しへ統一（ハードコード排除）。
- **完了条件**：
  - 8章の全項目が設定ファイルで変更できる。
  - 変更がゲームに反映される。
  - ハードコードが残っていない。

## Task018：アセット雛形整備

- **目的**：完成アセットの差し替え先を `assets/dinoparadise/` に用意する。
- **作業**：暫定モデル/テクスチャ/lang（日本語・英語）/必要音の雛形配置、原本は `blockbench/` `textures/` `animations/` に置く運用の確認。
- **完了条件**：
  - 全アイテム/ブロック/エンティティに名前が表示される（lang）。
  - 暫定でも描画が破綻しない。
  - 原本と完成品の置き場が分離されている。

## Task019：結合動作確認

- **目的**：体験フロー全体を通しで確認する。
- **作業**：「スポーン→討伐/巣で卵→設置→熱源で孵化→所有→給餌成長→成体→サドル→騎乗/インベントリ→ホイッスル命令→図鑑閲覧」を一連で検証。
- **完了条件**：
  - 上記フローがクラッシュ無しで完走する。
  - 各機能が `03_仕様書` と一致する。
  - 既知の不一致は `06_デバッグ報告` 用に記録される。
