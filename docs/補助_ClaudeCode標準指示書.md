# 補助_ClaudeCode標準指示書

> **役割**：Claude Code に作業を任せるときの「最初に渡す前提・守ってほしいこと」。
> セッション開始時にこの内容を伝えると、毎回ブレずに作業できる。

---

## セッション開始時に最初に伝える文（コピー用）

```
このフォルダは GitHub の <owner/repo> と連携済みのクローンです。
作業前に git pull、作業後に git add/commit/push まで自分で実行してください。

このリポジトリは docs/ の 02_開発ルール に従って作業します。
特に「リポジトリ構成」「仕様書にない機能を追加しない」「不明点は質問事項にまとめる」を守ってください。
```

---

## 守ってほしいこと（Claude Codeの基本ルール）

1. **前提資料を読む**：作業前に `docs/00_プロジェクト基盤定義.md`・`docs/02_開発ルール.md`、必要に応じて `03_仕様書.md`・`04_実装計画.md` を読む
2. **リポジトリ構成を守る**：`02_開発ルール` の構成どおりに配置（docs/ src/ assets/<modid>/ など）
3. **仕様外を作らない**：仕様書にない機能は追加しない。判断に迷う点は勝手に決めず「質問事項」として報告
4. **小さく進める**：1タスク＝1コミット。ビルド確認してから commit
5. **git運用**：作業前 pull、作業後 add/commit/push。コミットメッセージは簡潔に
6. **報告を残す**：実装後は `06_実装報告.md`、デバッグ後は `07_デバッグ報告.md` を作成

---

## 作業を頼むときのテンプレ（例：仕様書・実装計画づくり）

```
git pull。docs/00・01・02 を読み、02のルールに従って
docs/03_仕様書.md と docs/04_実装計画.md を作成。
仕様にない機能は追加せず、不明点は末尾に「質問事項」として一覧化。
完了したら git add docs && git commit && git push。
```

> アプリ（ダッシュボード）の各工程「プロンプトをコピー」が、この種の指示を自動生成する。
